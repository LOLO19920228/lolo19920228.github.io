    網頁切版直播班課前最終作業一助教建議

    .selfIntroduction 內的文字不需要再使用 div 包覆，樣式可以直接套用在最外層的 div
    .experience 的每個區塊內容建議可以使用 section 包覆，結構如以下示範 「Vito Evans」的電話及信箱可以增加 a 標籤，提高使用者的互動性，如以下示範 style 樣式都要寫在 CSS ，使用行內樣式的權重較高，後續程式碼撰寫會難以覆蓋
    body 內只需要保留以下樣式設定 text-decoration: none;、font-family: “Castoro”, serif;、line-height: 1.5; ，其餘的 class 樣式可以移出 body 外
    盒模型的設置可以寫成以下示範 *, *::before, *::after { box-sizing: border-box; }
    .footer li 不用設置高度
    以下回覆同學的問題：
    Q：為何 li 需要設定高度，否則會變為 72 px?
    Ans：會有這樣的情況是因為內層的 img 元素，img 是行內元素，行內元素的 vertical-align 預設是和父元素的 baseline 對齊的，而 baseline 本身又和父元素底邊有一定的間距。這個間距是來自於英文文字中像是 j, y 這些超過基準線的字符。
    因此要解決這個問題的話，同學可以將 img 的 display 設為 block，或是將 vertical-align 的設定修改為 top/bottom/text-top/text-bottom

    網頁切版直播班課前最終作業二助教建議

    可以嘗試減少選擇器階層數量，例如： .main .sidebar li a 改成 .sidebar a。
    header 的 .shoppingCar 和文字連結的 hover 樣式相同，可以使用同一個選擇器設定。
    .shoppingCar 不屬於表頭選單的一部分，可以移出 ul。
    承上，.shoppingCar 名稱有錯字，建議更改為 shoppingCart。
    sidebar 、產品列表與頁碼列表不需使用 d-flex flex-column ，因為內層排序的方向已經是由上至下。
    “甜點類別” 標題不是列表的內容，建議移出 ul>li 結構。
    承上，移出後，標題和 sidebar 列表可以再包一層 div 。 .main .productList .product .productInfo 不建議設定固定高度，建議使用 padding 推擠高度。
    「加入購物車」按鈕可以使用 button 標籤
    可以調整 pagination “上一頁” 與 “下一頁” 連結的 hover 效果。
    Q. 產品列表使用 gap:20px
    母層的justify-content-between就會失效無法靠到邊
    我目前的作法是sidebar往右硬擠 有更正確的作法嗎？
    Ans: 這邊的問題不是產品列表的 gap:20px 造成的，主要是因為 productList 的 flex 要排序第三個 li ，所以會撐滿剩下的 flex 空間，導致 productList 列表無法靠邊。
    調整方式可以對 productList 父層的 div 設定 width: 620px; 限制列表的寬度。
