Vue Cli

為什麼要用Vue Cli

  1. 前端開發日趨複雜，整合性的工具可以減少彼此開發上的差異
  2. 編譯環境越來越複雜，使用整合性工具建立環境
  3. 前後端分離形成主流
  4. 套件引用好管理

Vue Cli是什麼

  1. 基於Webpack所建置的開發工具
  2. 便於使用各種第三方套件（BS4、Vue Router等等）
  3. 可運行Sass、Bebel等編譯工具
  4. 獨特Vue檔案，一口氣包含html、js、css
  5. 便於開發SPA的網頁工具

Vue Cli開始流程

  1. 安裝NodeJs
  2. 建立Vue Cli全域環境

課程中用到的指令

  • 建立專案：vue create {{ 專案名稱 }}
  • 運行 Vue 開發環境：npm run serve
  • 編譯 Production 版本：npm run build
  • Nodejs 版本：node -v
  • npm 版本：npm -v
  • 回到資料夾頂端（win/mac）：cd\\ cd \\
  • 回到上一層（win/mac）：cd.. cd ..
  • 進入資料夾路徑：cd {‌{ 資料夾路徑 }}
  • 中斷目前操作：ctrl + c

安裝 Vue Cli 環境

Vue Cli環境中的檔案說明

  • node_modules：運行所需要的延伸套件
  • public：不需要編譯的檔案，可放較大的圖片與其他資源（除了index.html，編譯過後的檔案會與它做結合）
  • src
    • assets:較小的圖片、CSS、JS等等
    • components:頁面下掛載的子元件
    • router:路由表
    • views：獨立頁面需要出現的區塊
    • APP.vue：通常不會放除了<router-view/>以外的內容，是存放所有頁面都需出現的區塊，例如: 網站 logo / 導覽列 / 版權宣告
    • main.js:整個專案的進入點（全域的設定檔案、套件、路由表）
  • .browserslistrc：瀏覽器預期要支援的版本
  • .editorconfig：編輯器的設定檔
  • .eslintrc.js：檢視程式工具的設定檔與規則
  • .gitignore：使用git時要忽略的檔案
  • babel.config.js：babel設定檔案
  • package-lock.json：套件版本以及相關描述
  • package.json：專案的配置檔案（包含套件等等）
  • README.md：說明檔案
